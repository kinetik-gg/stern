# Kinetik UI Specification

This file is the canonical entrypoint for the Kinetik UI architecture specification. The architecture content is split into focused parts so each Markdown file stays small enough to review and maintain.

## Specification Parts

| Sections | File | Scope |
| --- | --- | --- |
| 1-10 | [Foundations](specs/01-foundations.md) | Purpose, philosophy, system overview, crate layout, terminology, frame lifecycle, coordinates, input, identity, and memory. |
| 11-14 | [Layout And Interaction](specs/02-layout-and-interaction.md) | Layout, dock/frame/panel responsibilities, interaction primitives, and the action system. |
| 15-22 | [Rendering, Text, Components, And Surfaces](specs/03-rendering-text-components.md) | Render primitives, renderer boundary, text, theme, components, collections, overlays, and viewport surfaces. |
| 23-28 | [Runtime, Platform, Accessibility, And Resources](specs/04-runtime-platform.md) | Non-blocking work, redraw scheduling, platform adapters, accessibility, resources, and diagnostics. |
| 29-34 | [Testing, CI, Showcase, Performance, And Workflow](specs/05-quality-workflow.md) | Testing strategy, CI, showcase application, performance requirements, implementation workflow, and open questions. |
